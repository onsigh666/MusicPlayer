## 规则 ##
1. 禁止一次性读取目录下所有文件，优先使用search_code检索相关逻辑；
2. 先通过list_files浏览目录结构，推理可能相关文件；
3. 明确目标函数/模块后再调用read_file；
4. 如果只需要了解函数作用，优先查找函数签名，不必读取全部实现；
5. 当信息足够完成修改时，停止继续读取代码。  
## 必做项 ##
1. 扫描歌曲，做成播放列表（要持久化）  
• 让用户选择一个文件夹，程序递归扫描其中的音频文件（至少支持.mp3），列在一个列表里，双击即可播放。  
• 扫描结果要持久化：把扫描到的曲库保存到文件，下次启动直接从文件加载、不必重新扫
描；用户可以再用"添加文件夹 / 重新扫描"来更新曲库。  
• 验收：扫描一次后关闭程序再打开，列表还在（没有重新扫描）；再添加一个文件夹，新歌
能并进来。
2. 基本播放控制
• 播放 / 暂停 / 停止 / 上一首 / 下一首 / 音量调节。  
• 一个进度条：随播放实时前进，且可以拖动跳转到任意位置。  
• 显示当前时间 / 总时长（如01:23 / 04:05 ）。  
• 验收：各按钮功能正常；拖动进度条能 seek；时间显示正确。  
3. 显示当前歌曲信息
• 从音频文件自身的元数据（标签）里读取并显示当前歌曲的标题（能一并读出歌手、专辑更
好）；确实没有标签时，才退回用文件名。   
• 验收：切歌时信息随之更新；对带标签的歌曲，显示的应是标签里的标题，而不是文件名。
4. 加载并同步歌词（本作业的重点）
• 自动寻找与歌曲同名的.lrc 歌词文件（如青花瓷.mp3 →青花瓷.lrc ）  
• 解析 .lrc ：把[mm:ss.xx]歌词 解析成"时间 → 歌词行"的有序列表。  
• 播放时，歌词逐行滚动，当前行高亮，并随播放进度自动切换。  
• 歌词时间微调（必做）：提供 + / − （如每次 ±0.5 秒），整体前移 / 后移歌词时间，用来校正"歌词和音乐对不齐"——.lrc 的时间常有误差。调整后立即生效，并建议把这个偏移量对该歌曲持久化（下次打开还记得）。  
• 验收：放一首带.lrc 的歌，歌词能跟着音乐一行行高亮走；用+/− 能把对不齐的歌词调到对齐。  
5. 歌词拖动滚动 + 跳转（做成音乐 App 那种交互）  
不是"点一行就跳"那么简单，要做成主流音乐 App 的歌词交互：
• 拖动浏览：用鼠标（或触摸板）在歌词区上下拖动，整段歌词跟着一起滚动，可以自由翻看
前后的歌词（此时不再自动跟随播放）。  
• 中间行的播放按钮：拖动 / 浏览时，屏幕中间那一行歌词旁边出现一个播放（跳转）按钮；
点它就跳转到这一行对应的时间并从那里继续播放。  
• 自动回弹：拖完之后，若在一定时间内（如 3 秒）没有点那个按钮，歌词自动滚回当前正
在播放的那一句，恢复"跟随播放"状态。  
• 验收：能拖动翻看歌词；中间行有跳转按钮、点了能跳到那句播放；拖完不点、过几秒自动
回到当前播放句。  
6. 播放模式
• 支持切换列表循环 / 单曲循环 / 随机播放三种模式，界面上能看出当前是哪种模式。  
• 一首放完后，按当前模式自动决定下一首（随机模式要真的随机、单曲循环要重放本曲）。  
• 验收：三种模式都能正确影响"自动播放下一首"的行为。  
7. 搜索 / 过滤
• 提供一个搜索框，输入关键字后，播放列表实时过滤出标题（或文件名）匹配的歌曲。  
• 验收：输入关键字，列表只剩匹配项；清空关键字，恢复全部。  
8. 转码  
• 选中一首歌，能把它转成另一种格式（例如flac → mp3 、wav → mp3 ），转码结果保存到指定位置。  
• 转码时界面不能卡死（用异步方式，比如
QProcess ），转码开始/结束要有提示，能看到进
度或状态。  
• FFmpeg 的使用方式见第二节（推荐
QProcess 调
ffmpeg.exe ）。  
• 验收：选一首歌点转码，能生成目标格式的可播放文件，过程中界面不卡。  
9. 界面美化（基础）  
• 用QSS给界面做一套基本样式：暗色背景、圆角、按钮 / 列表项的悬停效果，整体协调不
丑。  
• 验收：界面明显经过 QSS 美化，不是一堆默认灰控件堆在一起。
